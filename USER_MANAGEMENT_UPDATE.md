### 1. 数据清理

**问题**：教练员不应该同时拥有"普通用户"角色，因为教练员已经包含了普通用户的所有权限。

**解决方案**：
- ✅ 已清理数据库：移除了教练员的普通用户角色
- ✅ 更新后端逻辑：分配角色时，如果分配了教练员角色，自动移除普通用户角色

**清理结果**：
```
coachUser → 教练员（已移除普通用户角色）✅
```

## 🔧 后端逻辑改进

### 角色分配逻辑

**更新前**：
- 用户可以同时拥有多个角色
- 教练员可能同时拥有"教练员"和"普通用户"角色

**更新后**：
- 如果分配了教练员角色，自动移除普通用户角色
- 因为教练员已包含普通用户的所有权限

**代码实现**：
```java
// 如果分配了教练员角色，移除普通用户角色
boolean hasCoachRole = roleIds.contains(ROLE_COACH);
if (hasCoachRole) {
    roleIds.remove(ROLE_USER);
}
```

## 📊 角色权限关系

### 权限包含关系

```
SuperAdmin (超级管理员)
  └─ 拥有所有权限 ✅

Coach (教练员)
  ├─ user:read:self (查看自己的资料)
  ├─ user:update:self (编辑自己的资料)
  ├─ user:read:assigned (查看绑定用户)
  └─ user:update:assigned (编辑绑定用户)
  └─ 包含 User 的所有权限 ✅

User (普通用户)
  ├─ user:read:self (查看自己的资料)
  ├─ user:update:self (编辑自己的资料)
  └─ user:read:coach (查看教练资料)
```

**结论**：教练员已包含普通用户的所有权限，因此不需要同时拥有两个角色。

## 🎯 使用说明

### 1. 搜索用户

- **用户账号**：模糊搜索用户账号
- **用户名称**：模糊搜索用户名称
- **系统角色**：根据RBAC角色过滤用户
  - 超级管理员
  - 教练员
  - 普通用户

### 2. 分配角色

1. 点击"分配角色"按钮
2. 在弹窗中选择要分配的角色
3. **注意**：如果选择了"教练员"，系统会自动移除"普通用户"角色
4. 点击"确定"完成分配

### 3. 角色显示

- **超级管理员**：红色标签 🔴
- **教练员**：蓝色标签 🔵
- **普通用户**：绿色标签 🟢

## ⚠️ 注意事项

1. **角色互斥**：
   - 教练员和普通用户角色互斥
   - 分配教练员角色时，会自动移除普通用户角色

2. **权限继承**：
   - 教练员自动拥有普通用户的所有权限
   - 不需要同时分配两个角色

3. **数据一致性**：
   - 已清理历史数据中的重复角色
   - 后续分配角色时会自动处理
